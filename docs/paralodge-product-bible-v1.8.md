## 000 产品总定义

Paralodge / 平行公寓是一栋可以用手机网页进入的虚拟平行公寓。

用户打开链接，不是进入普通网站，不是进入商城，不是进入社区，也不是进入许愿墙。用户像深夜推开一栋公寓的门，进入 1F 愿力大厅，被阿青姐接待，再根据自己的状态，被愿力送到楼内不同空间。

核心体验：

# 推门进入
# 阿青姐接待
# 选择今晚想去哪里
# 进入对应空间
# 发愿 / 点灯 / 护灯 / 还愿 / 传灯
# 回到自己的房间或小店

世界观主句：

# 一座公寓，万家灯火

核心情绪：

# 欢迎，夜归人

## 001 产品第一性原理

Paralodge 首先要验证的不是功能完整，而是：

# 用户愿不愿意推门进入
# 用户能不能看懂下一步
# 用户愿不愿意留下一句话
# 用户愿不愿意点一盏灯
# 用户愿不愿意回来护灯
# 用户愿不愿意绑定 LINE
# 用户愿不愿意从虚拟空间走向现实服务

如果这些不成立，后面的 App、PWA、云端、支付、周边、商业街入驻都没有意义。

v1.8 的核心目标：

# 进入感
# 低学习成本
# 可回访
# 有归属
# 可转化

## 007 前台动作系统

v1.8 前台保留五个动作：

# 发愿
把心里的话交给这栋楼

# 点灯
第一次看见这个愿望，给它留一盏长明灯

# 护灯
这片灯火已经亮了，我再来护它一次

# 还愿
回来告诉这栋楼，路走到哪里了

# 传灯
把这点光留给后来的人

### 前台简化原则

首页不集中解释五个动作。用户只在对应场景看到当前动作：

# 没发愿时：发愿
# 第一次看到愿望时：点灯
# 已经点过灯时：护灯
# 自己的愿望有进展时：还愿
# 满足条件后：传灯

一句话原则：

# 用户只走路，系统算账

---

## 008 点灯规则

点灯是一次性动作。同一用户对同一对象只能点灯一次。

对象包括：

# 愿望
# 小店
# 记忆卡
# 房间愿望

点灯后，这盏灯成为长明灯。

前台文案：

# 你给这个愿望点了一盏灯

愿力计算：

给别人愿望点灯：

# 点灯者 +10
# 愿主 +10

给自己愿望点灯：

# 自己 +10

给小店点灯：

# 点灯者 +20
# 店主 +20

给自己小店点灯：

# 自己 +20

给别人记忆卡点灯：

# 点灯者 +10
# 记忆卡主人 +10

给自己记忆卡点灯：

# 自己 +10

---

## 009 护灯规则

护灯不是护某一盏具体灯。如果一个愿望有 100 盏灯，用户不需要知道自己护哪一盏。

护灯的真实含义是：

# 护这个愿望下的全部灯火
# 护这间小店的灯火
# 护这段记忆的灯火
# 护这个房间里的灯火

点灯增加灯数。护灯不增加灯数。护灯增加护灯次数和愿力反馈。

前台显示：

# 100 盏灯
# 23 次护灯

不显示：

# 第几盏灯被护住

护灯愿力：

护别人愿望：

# 护灯者 +10
# 愿主 +10

护自己愿望：

# 自己 +10

护小店：

# 护灯者 +10
# 店主 +10

护记忆卡：

# 护灯者 +10
# 记忆卡主人 +10

防刷：

# 同一用户对同一对象每 24 小时最多护灯一次

---

## 010 还愿规则

还愿不能只是按钮。还愿必须写一句记录：

# 这段路走到哪里了？

还愿类型：

# 已完成
# 还在路上
# 暂时放下
# 换一个愿望继续走

愿力：

# 还愿 +20

还愿不要求愿望完全实现。还愿是回来告诉这栋楼，自己走到哪里了。

---

## 011 传灯规则

传灯是最高等级动作。

传灯不是愿力转账。传灯不是愿力继承。传灯不是分数复制。传灯不是转发按钮。不存在“灯上积累的愿力值”这个规则。

传灯的意义：

# 一个走过一段路的人，把自己的灯留给后来的人

系统规则：

# 传灯生成 transmission_event
# 传灯者 +300 愿力
# 接灯者 +50 愿力
# 原灯进入「已传灯」状态
# 同一盏灯只能传一次
# 接灯者 7 天内不能再次传灯
# 只能传自己的灯

传灯者额外获得：

# 传灯记录卡
# 房间传灯记录
# 未来实体愿牌 / 纪念卡册故事加成

前台文案：

传灯前：

# 你已经走过这一段路
# 要不要把这盏灯传给后来的人？

按钮：

# 传一盏灯

传灯后：

# 这盏灯已经传给后来的人
# 愿力 +300

接灯者看到：

# 有人把一盏灯传给了你
# 愿力 +50

### 传灯触发条件

传灯按钮不常驻显示。满足任意一个条件才显示：

# 完成还愿
# 累计护灯 7 次
# 愿望收到 10 盏灯

这样传灯才有仪式感，不会变成普通按钮。

## 012 愿力规则

愿力不是钱。愿力不是虚拟币。愿力不是固定汇率积分。愿力不是排行榜分数。

愿力是：

# 用户在平行公寓里的行动痕迹
# 灯火记录
# 归来反馈
# 纪念权益依据

禁止：

# 100 愿力 = 1 日元
# 固定汇率
# 愿力提现
# 愿力交易
# 愿力转让
# 愿力投资
# 愿力排行榜

允许：

# 解锁数字愿卡样式
# 解锁房间灯光样式
# 解锁小店门灯样式
# 生成纪念编号
# 获得周边购买资格
# 申请系统活动折扣
# 获得系统赠送的数字纪念权益

折扣只能作为活动权益，不做固定兑换。

## 020 实体愿牌与周边

实体愿牌不是神秘承诺。不保证愿望实现。不宣传灵验。不做护身符。不做功德商品。

实体愿牌定义：

# 一张被看见、被点亮、被祝福过的愿望纪念牌

禁止文案：

# 加持
# 必灵
# 改命
# 能量
# 护佑
# 功德
# 购买获得好运
# 愿望一定实现
# 神秘力量

允许文案：

# 被看见

## 021 隐私与内容安全

# 愿望内容

默认匿名。  
不显示真实姓名。  
不显示联系方式。  
不显示住址、银行卡、身份证件。  
不展示伤害自己或他人的内容。  
不展示仇恨、骚扰、违法内容。

# 记忆墙

旧照片默认私密。  
公开展示不得出现清晰可识别人脸。  
不得未经同意上传他人可识别照片。  
不得展示身份证件、地址、电话。  
不得做祭祀化、灵验化、超自然承诺。

# 陪护室

不做医疗建议。  
不做心理诊断。  
不承诺治疗效果。  
不替代专业帮助。

# 商业街

不允许：

# 诈骗
# 投资收益承诺
# 医疗功效承诺
# 违法商品
# 假冒品牌
# 色情暴力
# 侵权内容
# 低俗广告
# 夸张疗效

页面出现：

# 发愿
# 点灯
# 护灯
# 还愿
# 传灯

传灯者 +300。接灯者 +50。同一盏灯只能传一次。接灯者 7 天内不能再次传灯。页面不出现“灯上积累的愿力值”。

## 025 README 摘要

# Paralodge / 平行公寓 v1.8

Paralodge 是一栋可以用手机网页进入的虚拟平行公寓。

当前版本是：

# 中文首发
# LINE 可选绑定
# 长明灯愿力系统
# 房间归属系统
# 商业街外链成交系统

核心动作：

# 发愿
# 点灯
# 护灯
# 还愿
# 传灯

核心空间：

# 愿力大厅
# 聚会间
# 商业街
# 公寓
# 陪护室
# 记忆墙

核心规则：

# 首页不做许愿墙
# 商业街不做商城
# 愿力不是钱
# 传灯不复制愿力
# 实体愿牌是纪念品
# 不做神秘承诺
# 不做排行榜
# 不做评论区
# 不做 AI 问答

最终定版一句话：

# Paralodge v1.8 不追求功能堆叠，而是让用户低压力进入、留下愿望、点亮灯火、回来护灯、完成还愿、把光传给后来的人，并让小店和记忆在不变成商城和玄学的前提下，走向现实服务。
